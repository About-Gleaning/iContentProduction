## 第一节：什么是 Markdown

**主持人 A**：
> Markdown 是一种轻量级标记语言，由 John Gruber 在 2004 年创建。

它的设计哲学是：

1. **易读易写**
2. **纯文本格式**
3. **可转换为 HTML**

**主持人 B**：没错！让我看一个简单的例子：

```markdown

**粗体** 和 *斜体*
- 列表项
```

## 第二节：实际应用

**主持人 A**：在开发中，Markdown 的应用场景非常广泛：

### 文档编写
- README 文件
- API 文档
- 技术博客

### 协作工具
- GitHub Issues
- Pull Request 描述
- Wiki 页面

**主持人 B**：我最常用的是写 README：

```markdown
# 项目名称

简短描述

## 安装

\`\`\`bash
npm install
\`\`\`

## 使用

\`\`\`javascript
const app = require('app');
\`\`\`
```

## 第三节：技巧分享

**主持人 A**：
> 💡 专业提示：使用一致的 Markdown 风格可以提高团队协作效率。

**主持人 B**：我推荐这些最佳实践：

1. **标题层级**
   - 一个文档只用一个 H1
   - 层级不要跳跃
   
2. **列表缩进**
   - 使用空格保持对齐
   - 保持一致的风格

3. **代码块**
   - 总是指定语言
   - 保持代码可运行
